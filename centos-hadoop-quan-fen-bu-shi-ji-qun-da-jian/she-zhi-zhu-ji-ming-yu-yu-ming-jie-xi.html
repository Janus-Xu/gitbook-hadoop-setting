设置主机名与域名解析

设置主机名与域名解析

  • 各个节点主机名修改

作用：Hadoop 集群中机器 IP 可能变化导致集群间服务中断，所以在 Hadoop 中最好以机器名进行配置

1.以 root 身份登录服务器节点，修改 /etc/sysconfig/network 文件，将 HOSTNAME=localhost.localdomain 改成 HOSTNAME=master

#这种方法是永久修改，必须重启才生效
#vim /etc/sysconfig/network

# Created by anaconda
NETWORKING=yes
HOSTNAME=hadoop-master

#临时修改，立刻生效
#hostname hadoop-master

//centos7修改主机名
#hostnamectl set-hostname hadoop-master

2.修改 /etc/hosts 文件，设置将 master 解析为本机 IP，在文件最后增加如下内容

#vim /etc/hosts

...
10.181.0.11     hadoop-master
10.181.6.112    hadoop-master-stby
10.181.9.43     hadoop-slave01
10.181.10.145   hadoop-slave02
10.181.10.72    hadoop-slave03
...

3.重启系统

#reboot

4.验证 hostname，结果为master，即为成功

[root@hadoop-master /root]#hostnamectl
   Static hostname: hadoop-master
         Icon name: computer-vm
           Chassis: vm
        Machine ID: 8d13a50988cc5c4972347415eddf7d47
           Boot ID: 0883c1d7c9e74d0f8c0daea050d7be55
    Virtualization: kvm
  Operating System: CentOS Linux 7 (Core)
       CPE OS Name: cpe:/o:centos:centos:7
            Kernel: Linux 3.10.0-514.21.1.el7.x86_64
      Architecture: x86-64

5.验证：如果能 ping 通，则证明修改成功

#ping hadoop-master

6.设置另外三台 DataNode 节点，分别做相同修改，步骤同上

节点名称 主机名称 HOSTNAME 内网IP
NameNode-Active hadoop-master 10.181.0.11
NameNode-Standby hadoop-master-stdy 10.181.6.112
DataNode hadoop-slave01 10.181.9.43
DataNode hadoop-slave02 10.181.10.145
DataNode hadoop-slave03 10.181.10.72

  • 各个节点域名解析

1.修改 hostname 完成之后，修改各个节点的 hosts 文件确保集群中的节点互相连通

在 master 节点上的 /etc/hosts 文件中，添加如下信息：

#vim /etc/hosts

...
10.181.0.11     hadoop-master
10.181.6.112    hadoop-master-stby
10.181.9.43     hadoop-slave01
10.181.10.145   hadoop-slave02
10.181.10.72    hadoop-slave03
...

2.测试是能否ping通

ping hadoop-master
ping hadoop-master-stby
ping hadoop-slave01
ping hadoop-slave02
ping hadoop-slave03

3.每个 DataNode 节点同 NameNode 节点，修改内容同上，这个配置文件集群中每台机器都一样
