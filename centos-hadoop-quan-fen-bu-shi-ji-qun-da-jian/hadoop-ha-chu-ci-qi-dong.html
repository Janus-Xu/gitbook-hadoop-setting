Hadoop HA 初次启动

Hadoop HA 初次启动顺序

1.启动 Zookeeper 集群 （hadoop-slave01、hadoop-slave02 和 hadoop-slave03）, 需要分别在集群中的每台服务器中启动。

$ ~/app/zookeeper-3.4.10/bin/zkServer.sh start

开启后使用 jps 查看是否有 QuorumPeerMain 进程，若有，则成功开启了，或者输入 zkServer.sh status 查看服务状态

正确的状态是一个 leader，两个 follower。

[hadoop@hadoop-slave01 /home/hadoop]$jps
11198 QuorumPeerMain
12779 Jps

[hadoop@hadoop-master /root]$~/app/zookeeper-3.4.10/bin/zkServer.sh status
ZooKeeper JMX enabled by default
Using config: /home/hadoop/app/zookeeper-3.4.10/bin/../conf/zoo.cfg
mkdir: cannot create directory  Permission denied
Mode: follower

2.启动 journalnode （hadoop-slave01、hadoop-slave02 和 hadoop-slave03），只需要在 hadoop-master 中

$cd ~/app/hadoop-2.7.4/sbin/
$./hadoop-daemons.sh start journalnode

启动成功后会多出一个 JournalNode 进程

[hadoop@hadoop-slave01 /home/hadoop]$jps
12229 JournalNode
12779 Jps
11198 QuorumPeerMain

3.格式化 HDFS （hadoop-master）

由于是将之前搭建的非 HA 集群转换成 HA 集群，所以不需要重新 format namenode ，如果是第一次新配置的集群，则在其中一台主节点上 (master) 执行 hadoop namenode -format 命令。（注意：该步骤由始至终只执行一次，执行多次可能造成集群启动不了）

hadoop namenode -format

格式化成功后，根据 core-site.xml 中的 hadoop.tmp.dir 配置生成的目录文件，这里配置的是 /data/hadoop/tmp，然后将其远程传输至另一台主节点 hadoop-master-stby 的 /data/hadoop/ 下。

scp -r /data/hadoop/tmp/ hadoop@hadoop-master-stby:/data/hadoop

4.启动 NameNode、同步备用 NameNode、启动备用 NameNode （hadoop-master 和 hadoop-master-stby）

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./hadoop-daemon.sh start namenode
#同步备用 NameNode 、启动备用 NameNode
[hadoop@hadoop-master-stby /home/hadoop/app/hadoop-2.7.4/bin]$./hdfs namenode -bootstrapStandby 
[hadoop@hadoop-master-stby /home/hadoop/app/hadoop-2.7.4/sbin]$./hadoop-daemon.sh start namenode

5.格式化 ZKFC （hadoop-master）

只需在 hadoop-master 一台机器上执行 hdfs zkfc -formatZK ，格式化 zkfc

（注意：该步骤只需在第一次配置 HA 架构时使用，成功后，集群日后的启动可掠过该步骤）

6.启动 DFSZKFailoverController 来监控 NameNode 状态 （hadoop-master 和 hadoop-master-stby）

在 hadoop-master 和 hadoop-master-stby 中都需要输入 hadoop-daemon.sh start zkfc 启动 zkfc ，同理，使用 jps 检验DFSZKFailoverController 进程是否存在，存在，则成功启动 zkfc 了。

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./hadoop-daemon.sh start zkfc  
starting zkfc, logging to /var/log/hadoop/hadoop-hadoop-zkfc-hadoop-master.out

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$jps
8554 DFSZKFailoverController
8604 Jps

7.启动 DataNode （hadoop-master）

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./hadoop-daemons.sh start datanode

8.启动 Yarn （hadoop-master 和 hadoop-master-stby）

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./start-yarn.sh 
[hadoop@hadoop-master-stby /home/hadoop/app/hadoop-2.7.4/sbin]$./yarn-daemon.sh start resourcemanager

9.查看进程服务

可以看出 hadoop-master 的 namenode 是处于一种 active 状态，那么 hadoop-master-stby 应该是处于standby 状态

[hadoop@hadoop-master /home/hadoop]$hdfs haadmin -getServiceState nn1
active
[hadoop@hadoop-master /home/hadoop]$hdfs haadmin -getServiceState nn2
standby

通过浏览器测试如下：

http://140.143.190.31:50070/           hadoop-master      active
http://140.143.184.98:50070/           hadoop-master-stby standby

在上述成功情况下，如果你去 hadoop-master 里面把这个 namenode 进程杀死的话，使用 jps 可以检测 namenode 进程不存在了，打开浏览器，会发现 hadoop-master-stby 自动切换为 Active，hadoop-master 变成无法连接。

[hadoop@hadoop-master /home/hadoop]$jps
25745 DFSZKFailoverController
28084 ResourceManager
25436 NameNode
9837 Jps

[hadoop@hadoop-master /home/hadoop]$kill -9 25436

[hadoop@hadoop-master /home/hadoop]$jps
25745 DFSZKFailoverController
28084 ResourceManager
10070 Jps

[hadoop@hadoop-master /home/hadoop]$hdfs haadmin -getServiceState nn1
17/11/08 08:22:26 INFO ipc.Client: Retrying connect to server: hadoop-master/10.181.0.11:9000. Already tried 0 time(s); retry policy is RetryUpToMaximumCountWithFixedSleep(maxRetries=1, sleepTime=1000 MILLISECONDS)
Operation failed: Call From hadoop-master/10.181.0.11 to hadoop-master:9000 failed on connection exception: java.net.ConnectException: Connection refused; For more details see:  http://wiki.apache.org/hadoop/ConnectionRefused
[hadoop@hadoop-master /home/hadoop]$hdfs haadmin -getServiceState nn2
active

重新启动 hadoop-master 的 namenode，hadoop-master 会变成 standby 状态

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./hadoop-daemon.sh start namenode
starting namenode, logging to /var/log/hadoop/hadoop-hadoop-namenode-hadoop-master.out

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$hdfs haadmin -getServiceState nn1
standby
[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$hdfs haadmin -getServiceState nn2
active

查看 yarn 状态

[hadoop@hadoop-master /root]$yarn rmadmin -getServiceState rm1
active
[hadoop@hadoop-master /root]$yarn rmadmin -getServiceState rm2
standby

http://140.143.190.31:8088/

输入：http://140.143.184.98:8088/ 会跳转到 http://hadoop-master:8088/cluster
