Hadoop HA 非初次启动

Hadoop HA 非初次启动和停止顺序

启动 HA 集群

1.启动 Zookeeper 集群 （hadoop-slave01、hadoop-slave02 和 hadoop-slave03）, 需要分别在集群中的每台服务器中启动。

[hadoop@hadoop-slave01 /home/hadoop/app/zookeeper-3.4.10/bin]$./zkServer.sh start
[hadoop@hadoop-slave02 /home/hadoop/app/zookeeper-3.4.10/bin]$./zkServer.sh start
[hadoop@hadoop-slave03 /home/hadoop/app/zookeeper-3.4.10/bin]$./zkServer.sh start

2.启动 HDFS （hadoop-master）

由于是非初次启动，启动 start-dfs.sh 的时候包含 namenode、datanode、journalnode 和 zkfc。

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./start-dfs.sh
Starting namenodes on [hadoop-master hadoop-master-stby]
hadoop-master-stby: starting namenode, logging to /var/log/hadoop/hadoop-hadoop-namenode-hadoop-master-stby.out
hadoop-master: starting namenode, logging to /var/log/hadoop/hadoop-hadoop-namenode-hadoop-master.out
hadoop-slave01: starting datanode, logging to /var/log/hadoop/hadoop-hadoop-datanode-hadoop-slave01.out
hadoop-slave03: starting datanode, logging to /var/log/hadoop/hadoop-hadoop-datanode-hadoop-slave03.out
hadoop-slave02: starting datanode, logging to /var/log/hadoop/hadoop-hadoop-datanode-hadoop-slave02.out
Starting journal nodes [hadoop-slave01 hadoop-slave02 hadoop-slave03]
hadoop-slave01: starting journalnode, logging to /var/log/hadoop/hadoop-hadoop-journalnode-hadoop-slave01.out
hadoop-slave03: starting journalnode, logging to /var/log/hadoop/hadoop-hadoop-journalnode-hadoop-slave03.out
hadoop-slave02: starting journalnode, logging to /var/log/hadoop/hadoop-hadoop-journalnode-hadoop-slave02.out
Starting ZK Failover Controllers on NN hosts [hadoop-master hadoop-master-stby]
hadoop-master: starting zkfc, logging to /var/log/hadoop/hadoop-hadoop-zkfc-hadoop-master.out
hadoop-master-stby: starting zkfc, logging to /var/log/hadoop/hadoop-hadoop-zkfc-hadoop-master-stby.out

3.启动 Yarn （hadoop-master 和 hadoop-master-stby）

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./start-yarn.sh 
starting yarn daemons
starting resourcemanager, logging to /home/hadoop/app/hadoop-2.7.4/logs/yarn-hadoop-resourcemanager-hadoop-master.out
hadoop-slave01: starting nodemanager, logging to /home/hadoop/app/hadoop-2.7.4/logs/yarn-hadoop-nodemanager-hadoop-slave01.out
hadoop-slave02: starting nodemanager, logging to /home/hadoop/app/hadoop-2.7.4/logs/yarn-hadoop-nodemanager-hadoop-slave02.out
hadoop-slave03: starting nodemanager, logging to /home/hadoop/app/hadoop-2.7.4/logs/yarn-hadoop-nodemanager-hadoop-slave03.out

[hadoop@hadoop-master-stby /home/hadoop/app/hadoop-2.7.4/sbin]$./yarn-daemon.sh start resourcemanager
starting resourcemanager, logging to /home/hadoop/app/hadoop-2.7.4/logs/yarn-hadoop-resourcemanager-hadoop-master-stby.out
停止 HA 集群

1.停止 HDFS（hadoop-master）

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./stop-dfs.sh
Stopping namenodes on [hadoop-master hadoop-master-stby]
hadoop-master: stopping namenode
hadoop-master-stby: stopping namenode
hadoop-slave01: stopping datanode
hadoop-slave02: stopping datanode
hadoop-slave03: stopping datanode
Stopping journal nodes [hadoop-slave01 hadoop-slave02 hadoop-slave03]
hadoop-slave01: stopping journalnode
hadoop-slave03: stopping journalnode
hadoop-slave02: stopping journalnode
Stopping ZK Failover Controllers on NN hosts [hadoop-master hadoop-master-stby]
hadoop-master: stopping zkfc
hadoop-master-stby: stopping zkfc

2.停止 Yarn （hadoop-master）

[hadoop@hadoop-master /home/hadoop/app/hadoop-2.7.4/sbin]$./stop-yarn.sh
stopping yarn daemons
stopping resourcemanager
hadoop-slave01: stopping nodemanager
hadoop-slave03: stopping nodemanager
hadoop-slave02: stopping nodemanager
no proxyserver to stop

[hadoop@hadoop-master-stby /home/hadoop/app/hadoop-2.7.4/sbin]$./yarn-daemon.sh stop resourcemanager
stopping resourcemanager

3.停止 Zookeeper 集群 （hadoop-slave01、hadoop-slave02 和 hadoop-slave03）

[hadoop@hadoop-slave01 /home/hadoop/app/zookeeper-3.4.10/bin]$./zkServer.sh stop
[hadoop@hadoop-slave02 /home/hadoop/app/zookeeper-3.4.10/bin]$./zkServer.sh stop
[hadoop@hadoop-slave03 /home/hadoop/app/zookeeper-3.4.10/bin]$./zkServer.sh stop
