Hadoop 安装与配置

Hadoop 安装与配置

1.在 hadoop-master 节点上使用 hadoop 用户下载 hadoop，目前是用的是 hadoop 2.7.4。

#mkdir ~/app
#cd ~/app
[hadoop@hadoop-slave01 /home/hadoop/app]#wget http://mirror.bit.edu.cn/apache/hadoop/common/hadoop-2.7.4/hadoop-2.7.4.tar.gz 

[hadoop@hadoop-slave01 /home/hadoop/app]#ls -la
total 260708
drwxr-xr-x 2 root   root        4096 Oct 21 17:11 .
drwx------ 6 hadoop hadoop      4096 Oct 20 23:50 ..
-rw-r--r-- 1 hadoop hadoop   266688029 Aug  6 16:15 hadoop-2.7.4.tar.gz

运行完上面的命令之后，hadoop-2.7.4.tar.gz 文件将会保存在/home/hadoop/app里面，请解压它，然后删除 hadoop-2.7.4 安装包

#tar -xzvf hadoop-2.7.4.tar.gz
#rm -rf hadoop-2.7.4.tar.gz

2.用 root 用户，在 /data 下创建 hadoop 目录，然后把 /data/hadoop 文件夹的读权限分配给 hadoop 用户，里面包含 dfs 和 tmp 两个目录，dfs 中包含 name 和 data 两个目录（配置中会指定这些目录），然后用如下命令保证该目录的所属和权限开放

#mkdir -p /data/hadoop/dfs/name
#mkdir -p /data/hadoop/dfs/data
#mkdir -p /data/hadoop/tmp
#设置所属
#chown -R hadoop:hadoop /data
#设置权限
#chmod -R 777 /data/hadoop

[root@hadoop-slave01 /data]#ls -la
total 36
drwxr-xr-x   4 hadoop hadoop  4096 Oct 21 00:02 .
dr-xr-xr-x. 20 root   root    4096 Oct 21 00:02 ..
drwxrwxrwx   4 hadoop hadoop  4096 Oct 20 23:58 hadoop
drwx------   2 hadoop hadoop 16384 Oct 12 18:08 lost+found

3.用 root 用户，修改 /tmp 文件夹的权限为 777，在所有节点均进行设置

#chmod 777 /tmp
[root@hadoop-slave01 /]#ls -la | grep tmp
drwxrwxrwx. 18 root root  4096 Oct 22 16:07 tmp

4.配置环境变量

  • 编辑 /etc/profile 文件
#vim /etc/profile
...
export HADOOP_HOME=/home/hadoop/app/hadoop-2.7.4
export PATH=$PATH:$JAVA_HOME/bin:$HADOOP_HOME/bin
...
  • 使环境变量生效

#source /etc/profile

  • 测试是否生效
[root@hadoop-master /root]#hadoop version

Compiled by kshvachk on 2017-08-01T00:29Z

5.编辑 hadoop-env.sh 文件，在里面找到JAVA_HOME，并将它的值设置为 JDK 所在的绝对路径

#su hadoop
#cd ~/app/hadoop-2.7.4/etc/hadoop/
#vim hadoop-env.sh
...
# The java implementation to use.
export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk-1.8.0.151-1.b12.el7_4.x86_64
...

默认情况下，Hadoop 生成的系统日志文件存放在 $HADOOP_HOME/logs 目录下，建议修改默认设置，使之独立于 Hadoop 的安装目录，便于 Hadoop 版本升级后安装路径发生变化，也不会影响日志文件位置。

切换 root 用户，在 /var/log/ 下创建 hadoop 目录，用于存放日志文件
#mkdir /var/log/hadoop/
设置所属
#chown -R hadoop:hadoop /var/log/hadoop/

#vim hadoop-env.sh
..
export HADOOP_LOG_DIR=/var/log/hadoop
...

6.编辑 core-site.xml

hadoop.tmp.dir 是 hadoop 文件系统依赖的基础配置，很多路径都依赖它。它默认的位置是在 /tmp/{$user} 下面，但是在 /tmp 路径下的存储是不安全的，默认的 /tmp 文件每次重新开机会被清空，与此同时 namenode 的格式化信息就会丢失。

#cd ~/app/hadoop-2.7.4/etc/hadoop/
#vim core-site.xml

<configuration>
        <property>
                <name>fs.defaultFS</name>
                <value>hdfs://hadoop-master:9000/</value>
        </property>
        <property>
                <name>hadoop.tmp.dir</name>
                <value>/data/hadoop/tmp</value>
        </property>
</configuration>

7.编辑 hdfs-site.xml

  • dfs.namenode.name.dir
    本地磁盘目录，NN存储fsimage文件的地方。可以是按逗号分隔的目录列表，fsimage文件会存储在全部目录，冗余安全。这里多个目录设定，最好在多个磁盘，另外，如果其中一个磁盘故障，不会导致系统故障，会跳过坏磁盘。由于使用了HA，建议仅设置一个。如果特别在意安全，可以设置2个

  • dfs.datanode.data.dir 本地磁盘目录，HDFS数据应该存储Block的地方。可以是逗号分隔的目录列表（典型的，每个目录在不同的磁盘）。这些目录被轮流使用，一个块存储在这个目录，下一个块存储在下一个目录，依次循环。每个块在同一个机器上仅存储一份。不存在的目录被忽略。必须创建文件夹，否则被视为不存在。

#cd ~/app/hadoop-2.7.4/etc/hadoop/
#vim hdfs-site.xml

<configuration>
        <property>
                <name>dfs.namenode.name.dir</name>
                <value>/data/hadoop/dfs/name</value>
        </property>

        <property>
                <name>dfs.datanode.data.dir</name>
                <value>/data/hadoop/dfs/data</value>
        </property>
        <!-- 指定 DataNode 存储 Block 的副本数量。默认值是3个，我们现在有3个 DataNode，该值不大于3即可。-->
        <property>
                <name>dfs.replication</name>
                <value>3</value>
        </property>
        <!-- 由于采用 HA 所以无需配置 SecondaryNameNode
        <property>
                <name>dfs.namenode.secondary.http-address</name>
                <value>node1:50090</value>
        </property>
        -->
</configuration>

8.编辑 mapred-site.xml，目录中只有一个 mapred-site.xml.template 文件，复制一个并重命名为 mapred-site.xml ，然后将文件修改为

#cd ~/app/hadoop-2.7.4/etc/hadoop/
#cp mapred-site.xml.template mapred-site.xml
#vi mapred-site.xml

<configuration>
        <property>
                <name>mapreduce.framework.name</name>
                <value>yarn</value>
        </property>
        <!-- Hadoop历史服务器
        <property>
            <name>mapreduce.jobhistory.address</name>
            <value>hadoop-master:10020</value>
        </property>
        <property>
            <name>mapreduce.jobhistory.webapp.address</name>
            <value>hadoop-master:19888</value>
        </property>
        -->
</configuration>

9.编辑 yarn-site.xml

#cd ~/app/hadoop-2.7.4/etc/hadoop/
#vim yarn-site.xml

<configuration>

<!-- Site specific YARN configuration properties -->
        <property>
                <name>yarn.resourcemanager.hostname</name>
                <value>hadoop-master</value>
        </property>
        <property>
                <name>yarn.nodemanager.aux-services</name>
                <value>mapreduce_shuffle</value>
        </property>

</configuration>

10.设置 pid 文件存储位置

基于java开发的程序，想要停止程序，必须通过进程 pid 来确定，而 hadoop 默认的情况下，会把 pid 文件存储在 Linux 上的 /tmp 目录的某个目录下，进程名命令规则一般是 框架名-用户名-角色名.pid，而默认情况下，linux 的 tmp 里面的东西，一天会删除一次，所以把 pid 文件放在这里面，并不是长久之计，为了安全起见，我们还是放到一个固定的目录下最好，当然不能放在 /tmp 下

#su hadoop
#mkdir -p /data/hadoop/pid

修改 hadoop-env.sh

#cd ~/app/hadoop-2.7.4/etc/hadoop/
#vim hadoop-env.sh
export HADOOP_PID_DIR=/data/hadoop/pid
export HADOOP_SECURE_DN_PID_DIR=/data/hadoop/pid

上述配置，影响 NameNode DataNode SecondaryNameNode 进程pid存储

修改 mapred-env.sh 

#vim mapred-env.sh
export HADOOP_MAPRED_PID_DIR=/data/hadoop/pid

上述配置，影响 JobHistoryServer 进程pid存储

修改 yarn-env.sh 

#vim yarn-env.sh
export YARN_PID_DIR=/data/hadoop/pid

上述配置，影响 NodeManager ResourceManager 进程pid存储
